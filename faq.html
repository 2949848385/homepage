技术问题

  • Q: 第一次加入服务器如何注册？以后如何登录？
  • A: 首次加入服务器需要游戏内注册，使用命令 /register [密码] [重复密码] 即可完成注册。例如： /register MyPassWord MyPassword 。以后每次加入游戏时，需要输入登录命令 /login [密码] 以登入游戏。在每次进入服务器时都会有提示，按提示操作即可。
  • Q: 如何设置名称前/后缀？
  • A: 请向管理员申请设置前/后缀。前/后缀可遵循游戏文字格式，具体代码请看 Minecraft Wiki
  • Q: Linux 下 Minecraft 如何输入中文（或其他CJK字符）？

  • A: Linux 下 MineCraft 输入中文/日文等解决方案：(无需Mod，无需修改客户端，支持任何输入法) 

    #!/bin/bash
xdotool key "t"
kdialog --inputbox ' ' | xclip -i -selection clipboard
xdotool key "ctrl+v"
xdotool key Return

    KDE以外用户请将第三行改成 gdialog --inputbox ' ' 2>&1 | xclip -i -selection clipboard ）

    （将以上脚本绑定到任意快捷键比如Super+t即可。在游戏中按下快捷键，会弹出对话框，即可输入任意文字，回车后自动发送。

    需要 xdotool 和 xclip 工具，请自行安装。如果是 apt-get 的话命令如下：

    sudo apt-get install xclip xdotool﻿

插件相关

  • Q: 如何使用锁箱插件？
  • A: 放置一个箱子(或其他可以存放物品的容器)，向其上放置一块木牌，不需要写任何内容，即可锁住该容器。只有木牌上有名字的玩家才可以打开。如果有些容器无法直接右键放置，请按住 Shift 再右键放置。
  • Q: 如何创建商店？
  • A: 玩家可以创建2种商店：销售商店与回收商店。创建商店使用以下步骤：
    1. 放置一个箱子(建议锁住)，其中放置一次性交易的物品(如一次销售一颗钻石则放置一颗钻石)，并使用红石粉 左键 点击该箱子以保存此箱子的位置与内容。
    2. 放置一个木牌(箱子与木牌可以在同一个世界的任何未受保护的相同或不同地方)，内容为：
    第一行| [Buy] (销售) 或 [Sell] (回收)
    第二行| 任意内容
    第三行| 任意内容
    第四行| 一次性交易价格
    完成木牌放置，使用 红石粉 左键 点击该木牌，即可完成商店的创建。其他玩家右键点击该木牌即可完成交易。
  • Q: 如何创建快递箱？如何向其他玩家发送快递？
  • A: 放置一个箱子(可以在任何位置，也可以上锁)，输入 /mailbox create 然后左键点击该箱子即可。请注意创造模式的玩家将不能使用该功能，请先将自己的游戏模式改回生存模式。
    发送快递时只需手中拿着需要发送的物品(可堆叠)，输入 /mailbox send [playername] 例如将一组火把发送给玩家 example 则拿着一组火把输入 /mailbox send example。发送快递将会收取快递费用，当前费用为每单快递1.0节操，每寄出1000米加0.5节操，寄往其他世界加0.2节操。

  • Q: 如何编辑世界商店数据？
  • A: 应如何编辑《世界商店数据》表格：

    这是面向不擅长使用Google云端表格服务的用户的一个以“您可以自己登记/更新店铺数据”为目的的教程。这个教程的内容主要是如何在Google云端编辑《世界商店数据》表格，会涉及到Google云端表格服务编辑的方式，但是不全面。

    首先您可以通过下方链接（选择《世界商店数据》，【打开】）来打开此列表。
    https://drive.google.com/folderview?id=0B1AQFr6H2Y27SzBnNlZuNEppS3c&usp=sharing

    如果您没有编辑权限，请在社群索要编辑权限。前提是您必须是已注册的活跃玩家。

    如何进行基础的编辑：

    添加新店铺条目：在左侧选择有您的店铺（您的单个SignShop商店）所在的商店（形成区域的多个店铺）的名称的行（点击那一行的左侧数字），然后右键，点击"在下方插入 1 行"，对这行的各个单元格（这一行的各个框）进行编辑直至符合要求。（如果没有您的店铺所在的商店请在末尾"在下方插入 1 行"，并在这一行的商店名称列单元格编辑商店名称，在这一行的其他单元格中补充店铺信息。）

    编辑单元格：选中单元格然后就可以使用您喜欢的方式修改其中的内容。

    删除店铺条目：点击需要删除的店铺条目的行的左侧（数字），右键，选择【删除行】。

    每个单元格的格式要求：

    单元格的格式要求目前与行无关，受到列的控制。

    商店名称（A）列：需要在在此商店中的第一个登记的店铺登记时添加在这个店铺的行（同时只在这一个店铺前添加，保持这个商店中的店铺都在这一行至下一个添加了商店名称的之前之间的行都是此商店的。），内容为商店的名称。然后添加备注（选中此单元格：菜单栏中【插入】->【备注】或使用快捷键Shift+F2），内容为商店在Minecraft世界中的坐标（可在Minecraft按【F3】）查看x,y,z对应）（主入口或内部的坐标），格式为：x:[x],y:[y],z:[z]（精确到小数点后一位。）

    销售内容（物品 x数量）（B）列：填写“[店铺销售的物品名称（以店铺所写信息为准）] x[物品数量]”（物品名称后有一个空格。）

    销售/收购（C）列：根据店铺运营方式填写“销售（向他人出售物资）”或“收购（向他人购买物资）”

    单次交易价（SignShop信息）（D）列：填写店铺单次交易物资的节操流通量，注意一定要以从SignShop获取的信息为准。（在Minecraft中对SignShop的木牌按下【按钮 1】获取信息）（有些时候不会与牌子设定信息相同。）

    所属玩家（E）列：填写维护此店铺的玩家的MinecraftUID或此服务器中的昵称。（可多填，“,”分割）

    商店状态（状态_mm/dd/yyyy）（F）列：填写："[商店记录时状态]_[记录时日期（格式：月月/日日/年年年年）]"状态分为正常（正常运营），缺货（物资不足单次交易），关停（暂时不运营）。（如果永久停止应删除店铺条目。）

    附加信息（G）列：任何店铺所有者想要添加的在此店铺条目其他单元格未表现出的合法信息。

    注：

    您无需调整任何单元格格式（字体，颜色，背景颜色，单元格大小等），此列表以配置完成可以进行自动调整。（在添加“单次交易价（SignShop信息）（D）列”的信息时也只需以数字方式填写价格，会自动补全单位与小数位数）
    快捷键[键1]+[键2]代表顺次按下[键1]，按下[键2]，放开两者。
    在您只要您所做的事情会有带来什么效果且不破坏任何此表格的信息及格式的情况下，您可以进行其他操作及操作方式来编辑。
    如果需要更多Google云端支持，请访问：https://support.google.com/drive/
    如果您还有任何其他问题可以向Hexadecimal或其他人询问。

  • Q: 如何成为高级玩家？
  • A: 成为高级玩家的判别标准是玩家贡献。此处的贡献分为以下几类。
    • 有类似于协助管理/建造工程/赞助服务器之类的贡献
    • 对某个或多方面的Minecraft能力擅长，例如建筑/利用游戏内容的设计
    • 能够与玩家熟悉，有一定的人气
    在以上任意一方面或多方面满足一定条件时即可申请升级为高级玩家。
